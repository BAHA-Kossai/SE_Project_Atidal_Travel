------------------------------------------------------------
Standard API Response Format
------------------------------------------------------------

All data returned from repository or route functions should follow this structure:

{
    status: "success" | "error"    // "success" if operation succeeded, "error" if there was a failure
    data: {                        // object containing the requested data, preserving original types
        ...
    }
    message?: "Error message if status is 'error'" // optional, only present on errors
}

Notes:
- The 'data' object should preserve native types from the database:
    - Numbers as numbers
    - Booleans as booleans
    - Strings as strings
    - Dates as ISO strings or date objects (depending on agreement)
- The 'message' field is only included on errors.
- This ensures consistency for all API consumers and simplifies front-end handling.

Example Success Response:
{
    status: "success",
    data: {
        id: 123,
        name: "Alice",
        balance: 12.99,
        isActive: true
    }
}

Example Error Response:
{
    status: "error",
    data: {},
    message: "User not found"
}
